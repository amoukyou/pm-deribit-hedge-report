第二章

认识风险，并筛出第一批值得监控的实盘问题

再探"明知山"

当我发现 PM 和 Deribit 的截止时间并不对等时，我知道很多人会在这里停下来。因为一旦时间不对齐，比较就不再完美，结构也不再工整，看起来像是一个"不值得继续"的问题。

但对我来说，这种感觉反而并不陌生。过去 12 年里，我在很多跨市场结构里做过类似的事：从最早俄罗斯、中国、日本、韩国之间的 BTC 价格差与山寨币三角套利，到后来数字货币 ETF 和加密股票之间的含量套利，每一次真正值得研究的机会，几乎都伴随着某种"不标准""不对等"或"不完全可比"的风险。

也正因为如此，我很少把这类问题理解成"这里有缺陷，所以不能做"，而更习惯把它理解成"这里有风险，所以值得继续研究"。因为很多时候，市场里真正隐藏利润的地方，本来就不是标准化最完美、所有人都一眼看懂的地方，而恰恰是那些存在错位、存在摩擦、存在不一致的地方。

核心风险：两边的时间对不上

PM 问题的结束时间由 PM 自己定，不完全统一。Deribit 标准期权到期时间相对固定：通常是 UTC 08:00。

这意味着：PM 问题结束的那一刻，和 Deribit 期权到期的那一刻，中间可能差几个小时甚至更多。在这段时间差里 BTC 价格可能变化，两边"看起来对称"的操作最终结果会出现偏差。

一句话：两边的钟不同步。

能不能找到更近的时间匹配？

Deribit 和 PM 固定差 8 小时。如果换其他交易所的期权，时间差能不能更小？

我们对比了 4 个有 BTC 期权的交易所，看它们的到期时间和 PM 16:00 UTC 的匹配度：

交易所 到期时间 与 PM 时间差 到期频率 数据来源
Deribit 08:00 UTC 8h（交易所先） 每日（工作日）+ 周五 + 月末 Deribit API
OKX 08:00 UTC 8h（交易所先） 每日（但日期覆盖较少） OKX API
Binance 08:00 UTC 8h（交易所先） 每日 + 周五 + 月末 Binance eAPI
IBIT（BlackRock BTC ETF） 20:00 UTC（4:00 PM ET） 4h（PM 先） 周一/周三/周五 CBOE API
发现：IBIT 期权在同日到期时只差 4 小时，比 Deribit 的 8 小时缩短一半。但 IBIT 是美股 ETF 期权，存在跨交易所资金调配成本（加密钱包 vs 美股券商）和行权价换算差异（IBIT 股价 vs BTC 价格）的问题。目前先不考虑 IBIT，继续用 Deribit 作为主要对冲工具。

两种最现实的处理方法

方法 1：事前筛选——挑时间接近的问题

一开始就尽量挑两边结束时间更接近的组合。只差一两个小时，风险就已经很小了。

方法 2：事后处理——主要机会兑现后提前平仓

如果 BTC 已经大幅偏离目标价（比如问 72,000 但已涨到 78,000），票已经接近 $1，继续持有只是白白暴露在时间差风险里。主动提前结束。

两种方法前后配合

方法 1 用在进入之前，方法 2 用在进入之后。

先把问题挑简单一点，再把退出时机掌握主动一点。

筛选标准

第一章筛出了 Above / Below 和 Price Range 两类最适合研究的问题。第二章在此基础上，按以下标准挑出观察样本：

• 两边结束时间尽量接近 • 规则简单 • 适合用上面两种方法处理

未解决的问题：哪边先平仓

从上面的样本可以看到，PM 和 Deribit 之间始终存在时间差。这意味着一定会面临一个问题：一侧先到期（或主动平仓），另一侧需要被动跟着平掉。

这就引出了流动性问题——当你一侧已经结算或平仓，需要立刻在另一侧平掉剩余头寸时，那边的盘口深度够不够？能不能按预期价格成交？

两种情况：

场景主动平仓侧被动平仓侧风险
Deribit 先到期Deribit 自动结算PM 需要手动卖出票PM 盘口深度不够 → 滑点
PM 先结束PM 自动结算Deribit 需要手动平仓期权Deribit 期权流动性不够 → 滑点

由于 Deribit 期权市场和 PM 的交易深度目前都不算很大，我们暂时无法确定应该选择哪边先平、哪边被动跟随。这需要在实际观察中，根据具体问题的盘口深度来判断。

这个问题留给第三章的实盘监控来验证。

第二章结论

研究项结论
核心风险 PM 16:00 UTC 结束，Deribit 08:00 UTC 到期，固定差 8 小时
其他交易所 OKX / Binance 同样 08:00 UTC（无改善）。IBIT 20:00 UTC 只差 4h，但跨交易所成本高，暂不考虑
应对方法 事前：挑同日到期的（Mar 17–20 最优）。事后：主要机会兑现后提前平仓
观察样本 从 PM API 抓取 45 个 Above + 11 个 Range = 56 个 BTC 实盘问题，全部匹配了 Deribit 期权合约
未解决 一侧到期后另一侧被动平仓的流动性问题——两边深度都不大，哪边先平尚无定论
