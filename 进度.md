# PM-Deribit 对冲套利研究

## 研究目标
利用 Deribit 期权对冲 Polymarket 加密市场的做市挂单，实现零方向风险敞口，赚取 PM maker rebate 及潜在的风险价差。

## 核心机制

### Polymarket Maker Rebate 规则
- **适用市场**：所有加密市场（5min / 15min / 1H / 4H / 日线 / 周线）
- **Rebate 来源**：Taker 手续费的 **20%** 按比例分配给 Maker
- **Taker 费率公式**：`fee_equiv = C × p × feeRate × (p × (1-p))^exponent`
  - Crypto: feeRate=0.25, exponent=2
  - 最大费率在 p=0.5 时 ≈ **1.56%**
- **分配方式**：按你的 fee_equivalent 占该市场总 fee_equivalent 的比例
- **支付**：每日 USDC 直接到钱包

### 费率曲线分析（Crypto 市场）
fee_equiv per share = p × 0.25 × (p(1-p))^2

| 价格 p | 单位费率 (占 notional %) | 说明 |
|--------|-------------------------|------|
| 0.10   | ~0.016%                 | 极低，不值得做 |
| 0.20   | ~0.10%                  | 低 |
| 0.30   | ~0.27%                  | 中 |
| 0.40   | ~0.46%                  | 高 |
| 0.50   | ~0.39%                  | fee_equiv 最高点 |

**最优做市区间：p ∈ [0.30, 0.60]**，fee_equivalent 权重最大。

## 策略设计

### 核心逻辑
1. **PM 端**：在加密二元市场（如 "BTC above $85000 on Mar 21?"）双边挂单做市
2. **Deribit 端**：用期权组合复制/对冲 PM 二元头寸

### PM Binary ↔ Deribit 期权映射
- PM binary "BTC > K at expiry" = Digital Call Option
- 复制方法：Deribit **Bull Call Spread**
  - 买 Call(K), 卖 Call(K+ΔK)
  - Payoff = min(max(S-K, 0), ΔK) / ΔK
  - ΔK 越小 → 越接近 digital → 但成本越高（pin risk）

### 三层收益来源
1. **Maker Rebate** (确定性)：被成交即拿，不依赖方向
2. **Bid-Ask Spread** (做市利润)：PM 上报价价差
3. **跨市场 Mispricing** (alpha)：PM 隐含概率 vs Deribit 隐含概率偏差

## 关键风险

| 风险 | 严重程度 | 缓解方案 |
|------|---------|---------|
| 基差风险 | 高 | 用紧 spread 复制，接受成本 |
| 期限匹配 | 中 | 优先做日线/周线市场，匹配 Deribit 到期日 |
| 对冲延迟 | 高 | 自动化 PM→Deribit 联动 |
| 资金效率 | 中 | 双边保证金锁定，需计算 ROE |
| Rebate 竞争 | 未知 | 需评估市场份额可行性 |
| 5min/15min 无法对冲 | 高 | 短期市场暂不参与，或用 perp 动态对冲 |

## 数学模型

### Digital Call 复制成本
```
复制价格 ≈ (C(K) - C(K+ΔK)) / ΔK
```
其中 C(K) 为 Deribit Call 价格。

### 套利条件
- 若 PM_price(YES) < Call_Spread_Cost → 买 PM YES + 卖 Deribit Spread
- 若 PM_price(YES) > Call_Spread_Cost → 买 PM NO + 买 Deribit Spread
- Edge = |PM_price - Replication_Cost| - 交易成本

### 年化 Rebate 收益估算
```
Daily Rebate = 日成交额 × avg_fee_rate × 20% × 市场份额
Annual ROE = Daily Rebate × 365 / 占用资金
```
